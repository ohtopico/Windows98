Descrição do produto

O Microsoft ActiveX Data Objects (ADO) é uma interface baseada na automatização para o acesso a dados. O ADO utiliza a interface OLE DB para acessar uma ampla variedade de fontes de dados incluindo, mas não se limitando a, dados fornecidos via ODBC.

Os usuários de RDO e DAO devem sentir-se rapidamente à vontade com a programação para ADO, pois o design geral do ADO é o resultado da nossa experiência no desenvolvimento destas interfaces.

Novos recursos

Nesta versão do ADO foram incluídos vários recursos para dar ao desenvolvedor mais poder e controle em seus aplicativos. (Consulte a documentação para obter mais informações sobre como utilizar estes recursos). Os novos recursos são os seguintes:

  • Integração com o Microsoft Client Cursors

    • Como no RDO 2.0, os projetistas agora têm um mecanismo de cursor do lado do cliente de alto desempenho que aceita atualização otimista de lotes, objetos Recordset desconectados e mais.

  • Comandos como métodos de conexão

    • Usando este recurso, os comandos associados a uma conexão se tornam métodos naquela interface de conexão.

  • Dados remotos

    • Usando este recurso, os usuários do ADO podem transmitir dados através de HTTP a um cliente, trabalhar com estes dados e enviá-los de volta para o servidor de HTTP.

Notas de instalação

Antes de executar o Programa de Instalação

Antes de instalar o ADO, certifique-se de que os computadores do seu cliente e servidor atendam os requisitos mínimos de hardware e software.

Requisitos de hardware

Para instalar o ADO, você precisa atender a certos requisitos de hardware que incluem o seguinte:

  • Qualquer computador compatível com x86 com Microsoft® Windows® 98 ou Microsoft® Windows NT®, ou computador baseado no processador Alpha da Digital Equipment Corporation.
  • Um disco rígido com no mínimo 14 megabytes (MB) de espaço disponível para a instalação completa de todos os componentes atualizados necessários para que o ADO seja executado.
  • Um mínimo de 12 MB de RAM para computadores clientes.
  • Um mínimo de 24 MB de RAM para computadores servidores (recomendados 32 MB de RAM).

Requisitos de software

Para que você possa instalar o ADO, seu computador deve atender os seguintes requisitos de software:

  • Qualquer uma das seguintes plataformas: Windows 98, Windows NT Workstation 4.0 ou posterior, ou Windows NT Server 4.0 (com Service Pack 3) ou posterior.

Para utilizar o ADO você precisará de um ambiente de programação que aceite objetos Automation. Atualmente isto inclui (mas não se limita a) produtos como o Microsoft® Internet Information Server 3.0 ou 4.0 (com Active Server Pages), os produtos contidos no Microsoft Office e o Microsoft Visual Studio™.

Problemas comuns

  • Quando estiver utilizando o método Open do objeto Connection do ADO no Microsoft® Visual C++® ou Microsoft® Visual J++™, ou outras linguagens que requerem que todos os parâmetros adicionais sejam especificados, você precisa especificar um quarto parâmetro como -1. Este parâmetro é um marcador para uma opção futura. Os usuários do Microsoft® Visual Basic® não precisam definir este parâmetro, pois ele será definido por padrão com o valor correto.


  • Se você estiver executando uma consulta JOIN em um banco de dados Oracle e solicitar um conjunto de teclas ou um cursor dinâmico, obterá um erro em vez de um cursor desatualizado. O Oracle só aceita cursores estáticos somente de leitura com associações.


  • Executar dois ou mais objetos Command na mesma conexão com um banco de dados do Microsoft® SQL Server causará um erro se algum dos objetos Command for um procedimento armazenado com parâmetros de saída. Se você estiver executando procedimentos armazenados do Microsoft SQL Server com parâmetros de saída, terá que utilizar conexões separadas para cada objeto Command, ou então desconectar os outros objetos Command da conexão (isto é, no Microsoft Visual Basic, Set cmd.ActiveConnection = Nothing).


  • Quando utilizar uma cláusula ORDER BY em uma consulta ao Microsoft SQL Server com o Microsoft SQL Server Sevice Pack 3 instalado, as colunas referidas na cláusula ORDER BY devem também ser ter referências na cláusula SELECT.


  • Quando você abrir um Recordset com a propriedade CursorLocation definida como adUseClient (i.e., um cursor do lado do cliente), a propriedade UnderlyingValue não estará disponível nos objetos Field do Recordset resultante.


  • No ADO 1.5 é possível copiar valores de um campo Recordset para outro usando uma linha de comando como:
    destRst.Fields("myfield") = srcRst.Fields("myfield")
    Foi observado que especificar de maneira explícita .Value dos dois lados da expressão melhora o desempenho e evita um problema conhecido como uma pequena insuficiência de memória nesta área. Deve-se utilizar a nova sintaxe: 
    destRst.Fields("myfield").Value  = srcRst.Fields("myfield").Value
    Isto deverá melhorar o desempenho e evitar a insuficiência de memória.
