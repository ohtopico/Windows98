Bem-vindo ao Personal Web Server 4.0

Bem-vindo ao Personal Web Server! No pacote do Personal Web Server, a Microsoft reuniu uma nova funcionalidade interessante, que transforma Microsoft® Windows 98® em uma plataforma ainda mais fácil de ser usada para compartilhamento de informações em um site pessoal da Web.

O pacote do Personal Web Server facilita a instalação de qualquer um dos seguintes novos recursos:

Microsoft Personal Web Server 4.0
Um servidor Web da área de trabalho que pode ser usado como host de um site da Web na intranet corporativa ou para desenvolver e testar um site da Web antes de atuar como host do site em um Provedor de serviços de Internet (ISP, Internet Service Provider).
Extensões do Microsoft FrontPage Server
Suporta o uso do Microsoft® FrontPage® para gerenciar seu site da Web, bem como criar o conteúdo do site.
Microsoft Transaction Server 2.0
Suporta a criação dos aplicativos do Microsoft® Transaction Server (MTS). Uma transação é uma operação de servidor realizada com êxito ou que apresenta falhas como um todo, mesmo que essa operação envolva várias etapas. O MTS também suporta o isolamento do processo dos aplicativos.
Microsoft Data Access Components 1.5
Facilita o uso de bancos de dados com suporte para o ActiveX™ Data Objects e para o driver Microsoft® Access.
Message Queue Server 1.0
Facilita a comunicação de programas aplicativos com outros programas aplicativos, de maneira rápida, confiável e assíncrona, enviando e recebendo mensagens. Os recursos chave do Message Queue Server (MSMQ), como o suporte para ActiveX, controles de segurança abrangentes, ferramentas de administração sofisticadas, conjunto de recursos extensivos e integração com produtos Microsoft estratégicos, como o Internet Information Server e o MTS, fazem do MSMQ o produto de enfileiramento de mensagens ideal para aplicativos executados em Windows 95, Windows 98 e Windows NT. O pacote do Personal Web Server inclui os clientes MSMQ dependentes e independentes.

Comparação de produto: PWS for Windows 98 e PWS for Windows NT Workstation

O PWS for Windows 98 dispõe de uma interface de administração gráfica projetada para usuários sem experiência anterior na criação e administração de sites da Web. Essa interface também é fornecida como a ferramenta de administração padrão no Personal Web Server for Microsoft® Windows NT® Workstation; entretanto, com a versão Windows NT Workstation, você também pode administrar seu site de publicação pessoal com o Internet Service Manager, a mesma ferramenta de administração com recursos completos usada para controlar o Microsoft Internet Information Server.

Para instalar o Personal Web Server

Consulte Resolução de problemas para obter maiores informações.

O CD do Windows® 98 inclui o Personal Web Server (PWS); entretanto, o PWS requer uma etapa de instalação em separado para que seja completamente instalado e configurado. Para instalar o PWS, você precisará do CD do Windows 98 e terá de seguir estas etapas:

  1. Insira o CD do Windows 98 na unidade apropriada.
  2. Clique em Iniciar e, em seguida, clique em Executar.
  3. Na caixa de diálogo Executar, digite x:\add-ons\pws\setup.exe
  4. Substitua a letra da unidade de CD por x e clique em OK.
  5. Siga as instruções na instalação do Personal Web Server.

Documentação do produto

Toda documentação do produto está disponível on-line. Você pode visualizar os tópicos de Ajuda clicando em Iniciar e apontando para o grupo de programas em que o produto está instalado ou usando o menu Ajuda da ferramenta de administração. A ajuda sensível ao contexto também estará disponível ao pressionar-se a tecla F1 durante a visualização de uma folha de propriedade ou caixa de diálogo.

Usando a documentação

Observe as seguintes orientações quando estiver usando a documentação on-line:

  • Se você clicar no ícone de impressão no quadro Conteúdo da documentação do produto 'Pacote de opções' e selecionar a opção Imprimir tudo que estiver contido no cabeçalho atual, é possível que verifique uma degradação temporária no desempenho do servidor se a seção selecionada contiver um grande número de tópicos. (Por este motivo, a opção Imprimir tudo que estiver no conteúdo não está disponível nesta versão.)

    • As páginas impressas com o ícone de impressão 'Conteúdo' não incluirão os documentos gráficos ou documentos vinculados. Além disso, o ícone de impressão 'Conteúdo' não funciona em conjunto com os exemplos de scripts e com os programas localizados na seção 'Amostras do desenvolvedor' da documentação do PWS. Ao selecionar um tópico na seção 'Amostras do desenvolvedor' e clicar no ícone de impressão 'Conteúdo', você receberá uma mensagem de erro indicando que não é possível localizar o arquivo.

    Uma forma alternativa de imprimir tópicos individuais é clicar com o botão direito do mouse em um tópico e, em seguida, clicar em Imprimir.

  • Se o Internet Explorer exibir uma mensagem de erro "Objeto HTTP 1.1/404 não localizado" quando você tentar visualizar a documentação, então o(s) arquivo(s) que você estiver tentando visualizar poderá(ão) não ter sido instalado(s)durante a instalação. Para instalar o(s) arquivo(s), execute novamente a 'Instalação' e escolha a opção de documentação apropriada.

  • O serviço WWW deve ser instalado e o servidor deve estar em execução para visualizar a documentação (exceto as observações sobre a versão e os arquivos de resolução de problemas). Se o site da Web for interrompido, ao clicar em Documentação do produto, você receberá uma mensagem de erro dizendo "Não foi possível estabelecer conexão com o servidor." Se você receber esta mensagem de erro, inicie o Personal Web Manager e verifique o status do site da Web; se tiver sido interrompido, então inicie o serviço.

  • A navegação da documentação ocorre, em maioria, no quadro esquerdo. Para visualizar o índice, clique no botão Conteúdo. Clique em um cabeçalho para visualizar as páginas que contêm sumários das informações incluídas nessa seção da árvore de conteúdo. Amplie e reduza as ramificações da árvore de conteúdo para visualizar ou ocultar seções de tópico.

  • Para visualizar o índice, selecione a folha de propriedade Índice no quadro esquerdo. Digite uma palavra para avançar para a parte do índice que estiver mais próxima da palavra digitada. Clique duas vezes em um tópico para verificar esse tópico ou uma lista de tópicos contendo a palavra-chave.

    Observação    Dependendo do navegador que você usar, os dispositivos de navegação do quadro esquerdo serão o ActiveX™ controles (no Internet Explorer) ou controles Java (nos navegadores Netscape); portanto, se você não visualizar o índice no quadro esquerdo, assegure-se de ter as seguintes configurações de navegador. Para o Internet Explorer, defina sua segurança de navegador como Média e ative os controles e plug-ins do ActiveX. Para os navegadores do Netscape, ative o Java e o JavaScript.

  • Algumas das funcionalidades disponíveis no quadro esquerdo da documentação, como o botão de sincronização do índice (TOC, table of contents synchronization button), poderão não funcionar adequadamente se houver uma versão anterior do Hhctrl.ocx instalada. Como o controle da versão deste arquivo não foi ativado na documentação, procure o Hhctrl.ocx e exclua quaisquer cópias deste arquivo localizadas na sua unidade de disco local. Da próxima vez em que acessar a documentação, você terá de aceitar o certificado que instalará o controle TOC atualizado.

Outros problemas e limitações conhecidos

A lista a seguir contém problemas e limitações já conhecidos nesta versão:

  • O Personal Web Server não suporta serviços FTP.
  • Se você estiver fazendo uma atualização a partir de uma versão anterior do PWS e tiver problemas na visualização do Default.asp, então provavelmente terá uma versão mais antiga desse arquivo. Para instalar a versão mais recente do arquivo do PWS 4.0, você deverá remover manualmente a versão antiga. A localização é C:\InetPub\Wwwroot.
  • Em alguns casos, os arquivos deixados atrás de uma instalação anterior das Extensões do FrontPage Server podem impedir que versões mais recentes do componente sejam instaladas corretamente. Se a Instalação do PWS reportar erros durante a instalação das Extensões do FrontPage Server, ou se o PWS, Microsoft FrontPage ou Microsoft Visual InterDev falharem na conexão com uma Web FrontPage, use o seguinte procedimento para corrigir o problema:
    1. Execute a Instalação do PWS, escolha Adicionar/Remover e desinstale as Extensões do FrontPage Server.
    2. Na pasta \Windows\System\, exclua os seguintes arquivos: Fp30txt.dll, Fp30utl.dll e Fp30wel.dll.
    3. Exclua a pasta \Program Files\Microsoft FrontPage\version3.0, incluindo todos os arquivos e subpastas.
    4. Execute a Instalação do IIS e reinstale as Extensões do FrontPage Server.
  • Os erros que ocorrerem durante a instalação não acompanhada serão gravados no Registro do Windows 98. Navegue até o caminho de Registro a seguir para obter informações sobre o erro:
    HKEY_LOCAL_MACHINE\
     Software
      \Microsoft
       \Windows
        \Instalação
         \OCmanager
          \Erros
  • A execução do Personal Web Server 4.0 em um computador com processador Intel 486 mais lento do que 50 MHz e que tiver 16 megabytes ou menos de RAM, aumenta a probabilidade de que os motores de script sejam degradados em um modo de compactação de heap de memória, processo que impedirá que o servidor aceite novas conexões. O processo de compactação de heap pode levar de 20 a 30 minutos para ser concluído (dependendo da velocidade do microprocessador).
  • Active Server Pages
    • Se o único banco de dados que você usar com o Active Server Pages® (ASP) for Microsoft SQL Server—isto é, você nunca usar os arquivos do Microsoft Access (.mdb) com o ASP—e criar instâncias do componente 'Acesso ao banco de dados' que você armazena no objeto Sessão , você poderá melhorar o desempenho alterando o modelo de encadeamento das instâncias do componente 'Acesso ao banco de dados' de Apartamento para Ambos no Registro. Para fazer a alteração de Apartamento para Ambos, clique duas vezes em Makefre15.bat na pasta 'Program Files\Common Files\System\Ado'.

      • Para reverter este processo, (isto é, para alterar o modelo de encadeamento de Ambos para Apartamento) clique duas vezes em Makeapt15.bat na pasta 'Program Files\Common Files\System\Ado'.

      Observação   Recomendamos que você não armazene instâncias do componente 'Acesso ao banco de dados' no objeto Sessão .

    • Há um erro na documentação do ASP que informa que o método MapPath não suporta sintaxe de caminho relativo, como ./path or ../path. O método MapPath suportará a sintaxe de caminho relativo se a propriedade de basemeta 'Ativar caminhos pai' estiver ativada. Quando a propriedade 'Ativar caminhos pai' não estiver ativada, usar o método MapPath com um caminho relativo resulta em erro.
    • O objeto 'Scripting.Dictionary' é erroneamente marcado como encadeado para Ambos. Deveria ser marcado como encadeado para Apartamento. Para alterar isso, use o Editor de Registro para abrir a seguinte chave de Registro:

      • HKEY_CLASSES_ROOT\
        CLSID
         \{EE09B103-97E0-11CF-978F-00A02463E06F}
          \InprocServer32

      Altere o valor nomeado de Modelo de encadeamento para Apartamento. Se você usar o objeto 'Dicionário' no escopo Aplicativo sem fazer essa alteração, poderá ocorrer danificação de dados.

    • Se você alterar as vinculações de rede em um site da Web em execução com usuários ativos, então esses usuários serão desconectados.
    • As propriedades definidas para os objetos ASP 'Solicitação', 'Resposta' e 'Servidor' só são válidas durante o processamento de uma solicitação. Por exemplo, se você definir uma propriedade em Global.asa, como Response.Buffer = True, a propriedade somente será aplicada a uma e não a todas as solicitações.
    • No momento, existem problemas conhecidos com a utilização do PerlScript com o ASP. Se você estiver executando o PerlScript a partir do ActiveWare, entre em contato com o ActiveWare para obter um novo motor para PerlScript.
    • A documentação 'Referência do scripter' enumera propriedades no objeto 'MyInfo' e informa que o Personal Web Server inicializa essas propriedades. Embora válidas para uma instalação em Macintosh do Personal Web Server no computador Macintosh, essas informações poderão não ser válidas para a instalação do Personal Web Server em Windows 95. Nenhuma das propriedades enumeradas na documentação 'Referência do scripter' existe quando MyInfo é a primeira instanciada. Em vez disso, essas propriedades são criadas durante a execução por meio de uma declaração de atribuição. Isto significa que, se não existir uma propriedade, a consulta desse valor retornará uma string vazia e qualquer tentativa de usar a propriedade em uma expressão resultará em erro.
    • Se você criar ou usar componentes desenvolvidos no Microsoft® Visual Basic 5.0™ (Edições Empresariais ou Profissionais) para uso com o ASP, recomendamos com insistência que atualize para o 'Pacote de serviços 2' do Visual Studio 97, para computadores de desenvolvimento e de servidor.

      • Você pode descarregar o 'Pacote de serviços 2' do Visual Studio 97 em http://www.microsoft.com/vstudio/sp/.

  • Ao executar componentes Java em um servidor ocupado, é possível que a máquina virtual java (VM, virtual machine) fique bloqueada durante a inicialização, como resultado de ter sido inicializada a partir de 40 ou mais segmentos. Isso ocorre somente na primeira vez em que o Java VM é inicializado por processo. Como um trabalho contínuo, adicione o seguinte código ao arquivo Global.asa para seu aplicativo (o "YourJavaObject" pode ser um componente provisório que não faz nada).
    • 
<SCRIPT LANGUAGE=vbscript  RUNAT=SERVER>

Sub Application_OnStart
	set initobj = Server.CreateObject("YourJavaObject")

End Sub

</SCRIPT>
  • Você somente pode publicar arquivos a partir de sua unidade de disco local. Por motivo de segurança, o assistente de publicação do Personal Web Manager não publicará um arquivo que residir em uma unidade de disco de rede.
  • No assistente de home page do Personal Web Manager, os componentes gráficos da folha de estilos associados aos modelos de home page são melhor visualizados com o Internet Explorer 4.01. São normais diferenças sutis entre a maneira como o modelo da home page aparece no IE 3.0 e no Netscape.
  • As versões 3.0 e 4.0 do Internet Explorer ignoram caracteres de quebra de linha em campos de texto. Para quebrar uma linha em uma entrada de campo de texto, adicione a marca <BR> HTML ao final de cada linha.
  • O ActiveX Data Objects (ADO) usa o método COM padrão de passar informações de erro, mas o ADO não verifica se existem outras informações no objeto 'ErrorInfor', apagando este objeto ao ser destruído. Existe um trabalho contínuo de código para este problema no Microsoft® Visual Basic®, Visual C++® e Visual J++™ que deve ser implementado na sua rotina de tratamento de erros.

    • Para o Visual Basic, o código deve ser lido:

    
ErrorHandler:
    ' cleanup
    If Not adoRS Is Nothing Then
	Set adoRS = Nothing
    End If
    If Not adoConn Is Nothing Then
	Set adoConn = Nothing
    End If
    
    Err.Raise Err.Number, "Bank.Accout.Post", Err.Description

Exit Function

    Para o Visual C++, o código deve ser lido:

    
//
// ErrorInfo is saved here because the following   
// ADO cleanup code may clear it.
//
IErrorInfo * pErrorInfo = NULL;
GetErrorInfo(NULL, &pErrorInfo);

if (adoRsBalance) adoRsBalance->Release();
if (adoCoConnection) adoCoConnection->Release();

AtlReportError( CLSID_CAccount, pErrMsg, IID_IAccount, hr);

//
// put the error back in TLS
//
SetErrorInfo(NULL, pErrorInfo);

    Para o Visual J++, o código deve ser lido:

    
if (adoRsBalance != null) {
	if (adoRsBalance.getState() == ObjectStateEnum.adStateOpen)
		adoRsBalance.Close();
	ComLib.release (adoRsBalance);
}

if (adoConn != null) {
	if (adoConn.getState() == ObjectStateEnum.adStateOpen)
		adoConn.Close();
	ComLib.release (adoConn);
}

    Observação   No Visual J++ você deve fechar explicitamente conjuntos de registros e/ou conexões, bem como liberar explicitamente os objetos ADO.

  • Quando o serviço Remote Data (RDS, Remote Data Service) 1.5 for instalado como um servidor no Windows 95 ou Windows 98 Beta  o uso do Personal Web Server impedirá que ele funcione. Isto será revisto em uma versão futura do serviço Remote Data. No momento, o RDS será executado com êxito a partir do Windows NT Server ou Workstation, em plataformas x86 ou Alpha.
  • Se você estiver gravando componentes do lado do servidor em Java e quiser usar o Microsoft®  Visual J++™ 1.1 para depurar seus componentes, adote o seguinte procedimento:
    1. Desenvolva arquivos .class de depuração para as classes Java que você quiser depurar.
    2. Assegure-se de que o aplicativo PWS que chama seu componente Java esteja configurado para ser executado no processo (a configuração padrão). Este mecanismo de depuração não funciona com aplicativos externos ao processo.
    3. No Visual J++, abra o projeto que contém suas classes Java, clique no menu Projeto, clique em 'Configurações' e, em seguida, clique na folha de propriedade 'Depurar'.
    4. Na lista Categoria, selecione Geral.
    5. Na caixa Classe para depuração/execução, digite o nome completo da classe (incluindo o nome do pacote) que você deseja depurar. Não inclua uma extensão de arquivo. Por exemplo, você pode digitar "IISSample.HelloWorld" como um nome de classe; o Visual J ++ é sensível a coincidir maiúsculas e minúsculas.
    6. Sob Depurar/Executar projeto sob, selecione Intérprete independente (somente aplicativos).
    7. Na lista Categoria, selecione Intérprete independente. Sob Intérprete independente, digite o caminho completo para o arquivo Inetinfo.exe em seu sistema. Por exemplo, se aplicável, você poderá digitar C:\Winnt\System32\Inetsrv\Inetinfo.exe, que é a localização padrão do Inetinfo.exe.
    8. Sob Argumentos do intérprete independente digite -e w3svc.

      9. Observação   Deixe a caixa Argumentos de programa em branco; caso contrário, o procedimento não funcionará.

    9. Clique em OK.
    10. no Visual J++, defina um ponto de interrupção na sua classe Java.
    11. No menu Desenvolver, escolha Iniciar depuração e, em seguida, clique em Ir.
      12. Esta etapa iniciará o Inetinfo.exe como um processo.

      Observação   Selecionar Executar <nome da classe> não iniciará a depuração.

    12. Depois que o depurador do Visual J++ tiver sido iniciado, chame sua classe Java abrindo o arquivo .asp que chama sua classe. Você disporá de aproximadamente 30 segundos para chamar seu código antes que o tempo do depurador do Visual do J++ expire. Depois que o depurador atingir o ponto de interrupção, o Visual J++ não mais imporá o limite de tempo esgotado; então você poderá prosseguir com a depuração de seu componente.
